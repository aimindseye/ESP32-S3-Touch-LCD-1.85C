# ESP32-S3-Touch-LCD-1.85C Rust diagnostics demo (esp-hal 1.0)

This project is a **Rust, bare-metal, esp-hal 1.0** diagnostics scaffold for the Waveshare **ESP32-S3-Touch-LCD-1.85C / 1.85C-BOX**.

It was updated using the uploaded vendor reference trees:

- `Arduino/examples/LVGL_Arduino/*`
- `ESP-IDF/ESP32-S3-Touch-LCD-1.85C-Test/main/*`

## What this project mirrors from the vendor demos

The uploaded Arduino and ESP-IDF demos both follow the same high-level init order:

1. `Flash_Searching`
2. `BAT_Init`
3. `I2C_Init`
4. `EXIO_Init` / `TCA9554PWR_Init(0x00)`
5. `PCF85063_Init`
6. `SD_Init`
7. `LCD_Init`
8. `Audio_Init`
9. `MIC_Speech_init` / `MIC_Init`
10. `LVGL_Init` and onboard/music UI

This Rust project mirrors that order **over serial diagnostics**, with the pieces that are solid in `esp-hal 1.0` implemented directly.

## What works in this Rust scaffold

- I2C bring-up on GPIO11/10
- TCA9554 probe and EXIO control on `0x20`
- CST816 touch reset via **EXIO1**, config reads, auto-sleep disable, and live touch coordinate reads on `0x15`
- PCF85063 RTC reads on `0x51`
- battery ADC sampling on GPIO8
- backlight GPIO heartbeat on GPIO5
- serial output that mirrors the vendor onboard diagnostics fields:
  - SD Card
  - Flash Size
  - Battery Voltage
  - RTC Time
  - Wireless Scan
  - Backlight Brightness

## What is intentionally not claimed as finished

This project does **not** yet include a full port of the vendor UI stack.

Still missing:

- full ST77916 QSPI LCD port
- LVGL UI pages
- SD/MMC runtime mount + capacity reporting
- Wi-Fi scan task
- PCM5101 audio playback
- ESP-SR speech recognition

Those parts are possible follow-on work, but they are not represented as complete in this package.

## Pin map used

### I2C / touch / RTC / EXIO
- SDA: GPIO11
- SCL: GPIO10
- Touch IRQ: GPIO4
- Touch reset: EXIO1
- LCD reset: EXIO2
- TCA9554: `0x20`
- CST816: `0x15`
- PCF85063: `0x51`

### Backlight / battery
- Backlight: GPIO5
- Battery ADC: GPIO8

### SD card
- CLK: GPIO14
- CMD: GPIO17
- D0: GPIO16
- SD CS helper line: EXIO3

### Display QSPI
- CS: GPIO21
- SCK: GPIO40
- D0: GPIO46
- D1: GPIO45
- D2: GPIO42
- D3: GPIO41
- TE: GPIO18

### Audio / mic
- PCM5101: BCLK=GPIO48, LRCLK=GPIO38, DOUT=GPIO47
- Mic path: BCLK=GPIO15, WS=GPIO2, DIN=GPIO39

## Build

This project targets the ESP32-S3 and uses `esp-hal = 1.0.0`. Because ESP32-S3 is an **Xtensa** target, you must use the Espressif Rust toolchain rather than plain host `stable`.

Typical steps on macOS/Linux:

1. install the Espressif Rust toolchain with `espup install`
2. source the environment file generated by `espup`:
   - `. $HOME/export-esp.sh`
3. connect the board over USB
4. build or flash with the `esp` toolchain

Example shape:
 - `. $HOME/export-esp.sh`
- `cargo +esp build --release`
- `espflash flash target/xtensa-esp32s3-none-elf/release/esp32s3-touch-lcd-1-85c-rust-diagnostics`

If you prefer not to type `+esp` every time, this repo now pins the correct toolchain in `rust-toolchain.toml`:

```toml
[toolchain]
channel = "esp"
components = ["rust-src"]
```

## Expected runtime behavior

On boot, the app prints:

- known board sizes
- init-step progress matching the vendor flow
- I2C probe results for `0x20`, `0x15`, `0x51`
- CST816 config values
- periodic onboard-style diagnostic summaries
- live touch coordinates when the screen is touched
